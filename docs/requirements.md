# Functional Requirements (MVP)

## Roles

1. Admin
2. Expert
3. Operator

## Main Features

1. User registration/login and JWT-based authentication.
2. Experiment creation and metadata management.
3. Input/import of experiment parameters.
4. Criteria and weights configuration.
5. Integral score calculation.
6. Risk level classification.
7. Recommendation generation.
8. Evaluation history view.
9. Basic reporting.

## Non-Functional Requirements

- Security: role-based access control, input validation, audit logging.
- Performance: API response under normal load should remain acceptable for interactive usage.
- Scalability: modular backend architecture and relational schema ready for expansion.
- Reliability: consistent data persistence and error handling.
